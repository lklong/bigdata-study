# HS2/HMS 竞态条件与并发问题集

> 目标版本：Hive 2.3.x / Hive 3.x | 共计 7+ 个问题点

---

## 一、问题总览

| 问题 | 位置 | 严重程度 | 影响 |
|------|------|----------|------|
| HiveSessionImpl 非线程安全 | HS2 Session | P1 | 并发查询数据错乱/NPE |
| SQLOperation 跨线程复用 SessionState | HS2 Operation | P1 | SessionState 污染 |
| Hive.java 分区创建竞态 | QL Metadata | P2 | 分区创建失败/重复 |
| ObjectStore 重名竞态 | HMS Server | P2 | 元数据名称冲突 |
| ZK 锁管理器父节点竞态 | QL Lock | P2 | 锁操作异常 |
| DbTxnManager 空指针竞态 | QL TxnManager | P2 | NPE 导致查询失败 |
| DriverTxnHandler 竞态 | QL Driver | P2 | 事务状态不一致 |
| Session close 多线程竞态 | HS2 Session | P1 | 事务泄漏(HIVE-29286) |

## 二、详细分析

### RC-001 HiveSessionImpl 非线程安全

- **严重程度**: P1
- **影响版本**: Hive 2.3.x、Hive 3.x
- **涉及文件**: `service/src/java/org/apache/hive/service/cli/session/HiveSessionImpl.java:102-107`
- **错误堆栈特征**: 并发查询时出现 `NullPointerException`、`ConcurrentModificationException`、数据错乱
- **根因分析**:
  代码注释坦承 SessionState 内部非线程安全：
  ```java
  // HiveSessionImpl.java:102-107
  // TODO: some SessionState internals are not thread safe. The compile-time internals are synced
  //       via session-scope or global compile lock. The run-time internals work by magic!
  //       They probably work because races are relatively unlikely and few tools run parallel
  //       queries from the same session.
  //       1) OperationState should be refactored out of SessionState, and made thread-local.
  //       2) Some parts of session state, like mrStats and vars, need proper synchronization.
  ```
  编译时通过编译锁同步，但运行时"靠运气"——因为竞态不太可能发生且很少有工具在同一 session 中并行查询。
- **因果链**: `同 session 多线程查询` → `SessionState 非线程安全` → `变量/状态互相污染` → `NPE/错误结果`
- **社区状态**: 代码中标记为 TODO，**从未修复**（从 Hive 2.x 延续至今）
- **参数规避**:
  ```xml
  <!-- 禁止同 session 并行操作 -->
  <property>
    <name>hive.server2.parallel.ops.in.session</name>
    <value>false</value>  <!-- 强烈建议设为 false -->
  </property>
  ```
- **验证方法**: 在同一 session 中并发执行多个查询，观察是否有 NPE 或结果异常

### RC-002 SQLOperation 跨线程复用 SessionState

- **严重程度**: P1
- **影响版本**: Hive 2.3.x、Hive 3.x
- **涉及文件**: `service/src/java/org/apache/hive/service/cli/operation/SQLOperation.java:321`
- **错误堆栈特征**: 异步查询结果中出现其他查询的变量或状态
- **根因分析**:
  ```java
  // SQLOperation.java:321
  // TODO: can this result in cross-thread reuse of session state?
  SessionState.setCurrentSessionState(parentSessionState);
  ```
  子线程直接设置父线程的 SessionState 作为当前 SessionState。如果多个异步操作并行执行，可能导致 SessionState 跨线程污染。
- **因果链**: `setCurrentSessionState(parentSessionState)` → `多子线程共享同一 SessionState` → `ThreadLocal 变量跨线程互见` → `状态污染`
- **社区状态**: 代码中标记为 TODO，**未修复**
- **参数规避**: 同 RC-001，设置 `hive.server2.parallel.ops.in.session=false`

---

### RC-003 Hive.java 分区创建/移动竞态

- **严重程度**: P2
- **影响版本**: Hive 2.3.x、Hive 3.x
- **涉及文件**: `ql/src/java/org/apache/hadoop/hive/ql/metadata/Hive.java:2997,3045,3883,5778`
- **错误堆栈特征**:
  ```
  AlreadyExistsException: Partition already exists
  FileNotFoundException: File does not exist (分区目录)
  ```
- **根因分析**:
  多处代码注释标记了竞态条件：
  ```java
  // Hive.java:2997
  // created it (race condition).
  // Hive.java:3045
  // created it (race condition).
  // Hive.java:3883
  // This means the exception was caused by something other than a race condition
  // Hive.java:5778
  // harder to make race condition proof.
  ```
  并发创建同一分区时，两个线程可能同时检查分区不存在，然后同时尝试创建，导致 `AlreadyExistsException`。代码中有部分 catch 和重试逻辑，但不完善。
- **因果链**: `并发 INSERT 同分区` → `check-then-act 竞态` → `AlreadyExistsException` → `插入失败或重复`
- **参数规避**:
  - 使用动态分区时加分布式锁：`hive.support.concurrency=true`
  - 避免多个作业同时写入同一分区
- **验证方法**: 并发执行多个 INSERT 到同一新分区

### RC-004 ObjectStore 重名竞态

- **严重程度**: P2
- **影响版本**: Hive 3.x
- **涉及文件**: `standalone-metastore/metastore-server/src/main/java/org/apache/hadoop/hive/metastore/ObjectStore.java:5798`
- **根因分析**:
  ```java
  // ObjectStore.java:5798
  // NB: The below code can result in race condition where duplicate names can be generated (in theory).
  ```
  在生成唯一名称时存在理论上的竞态条件。虽然概率极低，但在超高并发下可能生成重复名称。
- **社区状态**: 代码注释标记为已知问题，未修复
- **参数规避**: 无直接参数，概率极低

---

### RC-005 ZooKeeper 锁管理器竞态

- **严重程度**: P2
- **影响版本**: Hive 2.3.x（使用 ZK 锁时）、Hive 3.x
- **涉及文件**: `ql/src/java/org/apache/hadoop/hive/ql/lockmgr/zookeeper/ZooKeeperHiveLockManager.java:558,562`
- **根因分析**:
  ```java
  // ZooKeeperHiveLockManager.java:558
  // or in a race condition where parent has already been deleted by other process when it
  // ZooKeeperHiveLockManager.java:562
  // can happen in a race condition where another process adds a zLock under this parent
  ```
  父 znode 删除与子 znode 添加之间存在竞态。一个进程删除父节点时，另一个进程可能正在添加子锁节点。
- **参数规避**: 使用 `DbTxnManager`（Hive 3.x 默认）代替 `ZooKeeperHiveLockManager`
- **社区状态**: Hive 3.x 默认使用 DbTxnManager，ZK 锁管理器已弱化

---

### RC-006 DbTxnManager/DriverTxnHandler 空指针竞态

- **严重程度**: P2
- **影响版本**: Hive 3.x
- **涉及文件**:
  - `ql/src/java/org/apache/hadoop/hive/ql/lockmgr/DbTxnManager.java:769`
  - `ql/src/java/org/apache/hadoop/hive/ql/DriverTxnHandler.java:614`
- **根因分析**:
  ```java
  // DbTxnManager.java:769
  // To prevent NullPointerException due to race condition, marking it as synchronized.
  // DriverTxnHandler.java:614
  // might call the below method concurrently. To prevent a race condition, marking it as synchronized.
  ```
  已通过 `synchronized` 关键字修复，但说明这些路径在设计上存在并发访问的可能。
- **社区状态**: 已通过 `synchronized` 修复
- **参数规避**: 无需额外配置

---

### RC-007 Session Close 多线程竞态导致事务泄漏 (HIVE-29286)

- **严重程度**: P1
- **影响版本**: Hive 4.2.0+（但根因在 3.x 同样存在）
- **涉及文件**: HiveSessionImpl（多个方法）
- **错误堆栈特征**: `NullPointerException` in SessionState access; 事务一直处于 OPEN 状态
- **根因分析**:
  当一个客户端 C2 正在等待编译锁，另一个客户端 C3 关闭同一 session S2 时：
  1. C3 成功关闭 S2，清除 SessionState
  2. C2 获得编译锁后访问已清除的 SessionState → NPE
  3. 如果涉及 ACID 表，C2 打开的事务无法正常关闭 → 事务泄漏
- **社区状态**: **HIVE-29286** (Open) — 2025年10月报告，尚未修复
- **参数规避**:
  - 避免多客户端/线程共享同一 HS2 session
  - 设置 `hive.server2.parallel.ops.in.session=false`

## 三、并发问题诊断命令

```bash
# 检查死锁
jstack $(pgrep -f HiveServer2) | grep -A20 "Found one Java-level deadlock"

# 检查 BLOCKED 线程及持有的锁
jstack $(pgrep -f HiveServer2) | grep -B2 -A10 "BLOCKED"

# 检查 SessionState 相关 NPE
grep "NullPointerException" /var/log/hive/hiveserver2.log | grep -i "session\|state" | tail -20

# 检查分区创建竞态
grep "AlreadyExistsException\|Partition already exists" /var/log/hive/hiveserver2.log | tail -20

# 检查泄漏的事务
beeline -e "SHOW TRANSACTIONS;" | grep OPEN | awk '{print $1, $2, $3}'
```

## 四、推荐配置

```xml
<!-- 防止并发竞态的核心配置 -->
<property>
  <name>hive.server2.parallel.ops.in.session</name>
  <value>false</value>
</property>
<!-- Hive 3.x 使用 DbTxnManager 代替 ZK 锁 -->
<property>
  <name>hive.txn.manager</name>
  <value>org.apache.hadoop.hive.ql.lockmgr.DbTxnManager</value>
</property>
```
