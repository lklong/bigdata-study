## 二、详细分析

### AUTH-001 Kerberos Ticket 过期导致连接失败

- **严重程度**: P1
- **影响版本**: Hive 2.3.x、Hive 3.x
- **涉及文件**:
  - `service/src/java/org/apache/hive/service/auth/KerberosSaslHelper.java`
  - `service/src/java/org/apache/hive/service/auth/HiveAuthFactory.java`
- **错误堆栈特征**:
  ```
  javax.security.sasl.SaslException: GSS initiate failed
  Caused by: GSSException: No valid credentials provided (Mechanism level: Failed to find any Kerberos tgt)
  org.apache.thrift.transport.TTransportException: SASL authentication failed
  ```
- **根因分析**:
  Kerberos ticket 默认有效期通常为 24 小时（可续期 7 天）。如果 HS2/HMS 的 keytab 配置不正确或续期机制失效：
  1. **HS2 端**：HS2 使用 keytab 自动续期，如果 keytab 文件权限不正确或 KDC 不可达，续期失败
  2. **HMS 端**：HMS 使用的 keytab 过期后，所有新的 Thrift 连接都会认证失败
  3. **客户端**：Beeline/JDBC 客户端的 TGT 过期后无法建立新连接
- **因果链**: `Kerberos ticket 过期` → `SASL 认证失败` → `Thrift 连接被拒绝` → `查询/DDL 操作全部失败`
- **参数规避**:
  ```xml
  <!-- HS2 Kerberos 配置 -->
  <property>
    <name>hive.server2.authentication</name>
    <value>KERBEROS</value>
  </property>
  <property>
    <name>hive.server2.authentication.kerberos.principal</name>
    <value>hive/_HOST@REALM</value>
  </property>
  <property>
    <name>hive.server2.authentication.kerberos.keytab</name>
    <value>/etc/security/keytabs/hive.service.keytab</value>
  </property>

  <!-- HMS Kerberos 配置 -->
  <property>
    <name>hive.metastore.sasl.enabled</name>
    <value>true</value>
  </property>
  <property>
    <name>hive.metastore.kerberos.principal</name>
    <value>hive/_HOST@REALM</value>
  </property>
  <property>
    <name>hive.metastore.kerberos.keytab.file</name>
    <value>/etc/security/keytabs/hive.service.keytab</value>
  </property>
  ```
  **关键排查项**：
  ```bash
  # 检查 keytab 文件
  klist -kt /etc/security/keytabs/hive.service.keytab
  
  # 检查当前 ticket
  klist
  
  # 检查 KDC 连通性
  kinit -V -kt /etc/security/keytabs/hive.service.keytab hive/hostname@REALM
  
  # 检查时钟偏移（Kerberos 要求 <5 分钟）
  ntpstat || chronyc tracking
  ```
- **验证方法**: `klist` 查看 ticket 有效期；日志检查 `GSS initiate failed`

### AUTH-002 MetaStorePlainSaslHelper Hack 类加载

- **严重程度**: P3
- **影响版本**: Hive 2.3.x、Hive 3.x
- **涉及文件**: `standalone-metastore/metastore-common/src/main/java/org/apache/hadoop/hive/metastore/MetaStorePlainSaslHelper.java:54-57`
- **根因分析**:
  ```java
  // MetaStorePlainSaslHelper.java:54-57
  // This is a hack. This method does nothing but is used to load the class and thus
  // We don't need this hack in HS2, but we do not know the code path through which
  // this class is loaded.
  ```
  使用空方法触发类加载，这是一个代码质量问题而非运行时 bug。说明 SASL 认证的初始化路径不够清晰。
- **社区状态**: 代码中标记为 hack，未有清理计划
- **影响**: 不影响运行时功能，仅影响代码可维护性

### AUTH-003 SAML 认证仅支持 HTTP-Redirect Binding

- **严重程度**: P3
- **影响版本**: Hive 3.x（SAML 是 3.x 新增功能）
- **涉及文件**: `service/src/java/org/apache/hive/service/auth/saml/HiveSaml2Client.java:50-52`
- **根因分析**:
  ```java
  // HiveSaml2Client.java:50
  // HTTP-Redirect binding. //TODO: Add support for HTTP-Post binding for SAML request.
  // HiveSaml2Client.java:52
  // //TODO Note that this implementation is only to be used when HiveServer2 is not
  ```
  HS2 的 SAML 认证目前只支持 HTTP-Redirect binding。大型企业 IdP（如 ADFS、Okta）通常推荐使用 HTTP-Post binding，当前实现不支持。
- **社区状态**: 代码中标记为 TODO，未修复
- **参数规避**: 如需使用 SAML，确认 IdP 支持 HTTP-Redirect binding；否则考虑使用 LDAP 或 Kerberos

## 三、认证问题诊断命令

```bash
# ===== Kerberos 诊断 =====

# 检查 keytab 文件内容
klist -kt /etc/security/keytabs/hive.service.keytab

# 测试 keytab 认证
kinit -V -kt /etc/security/keytabs/hive.service.keytab hive/$(hostname -f)@REALM

# 查看当前 ticket 状态
klist -e

# 检查时钟偏移
date && ssh <kdc_host> date

# 检查 NTP 同步
ntpstat 2>/dev/null || chronyc tracking 2>/dev/null

# 检查 HS2 SASL 认证日志
grep -i "sasl\|kerberos\|GSS\|authentication failed" /var/log/hive/hiveserver2.log | tail -20

# 检查 HMS SASL 认证日志
grep -i "sasl\|kerberos\|GSS\|authentication failed" /var/log/hive/hive-metastore.log | tail -20

# ===== 连接测试 =====

# Kerberos 模式 Beeline 连接
beeline -u "jdbc:hive2://<host>:10000/default;principal=hive/_HOST@REALM"

# HTTP 模式 Kerberos 连接
beeline -u "jdbc:hive2://<host>:10001/default;transportMode=http;httpPath=cliservice;principal=hive/_HOST@REALM"
```

## 四、Kerberos 常见故障排查表

| 错误信息 | 可能原因 | 排查命令 |
|----------|----------|----------|
| `GSS initiate failed` | Ticket 过期/KDC 不可达 | `klist; kinit -V -kt <keytab> <principal>` |
| `No valid credentials provided` | Keytab 不匹配/不存在 | `klist -kt <keytab>; ls -la <keytab>` |
| `Clock skew too great` | 时钟偏移 >5 分钟 | `ntpstat; date` |
| `Server not found in Kerberos database` | Principal 配置错误 | 检查 `_HOST` 是否正确解析为 FQDN |
| `Connection refused` | HMS/HS2 未启动/端口错误 | `netstat -tlnp \| grep <port>` |
| `SASL authentication failed` | SASL 配置不匹配 | 检查两端 `sasl.enabled` 一致 |
