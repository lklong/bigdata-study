## 一、课前导入（为什么要学这个？）

### 生活类比：快递分拣中心

想象一个快递分拣中心正在处理双十一的包裹。每个包裹（数据行）需要按目的地省市（分区列）分拣到不同的货架（分区目录）。静态分区=提前准备好货架；动态分区=包裹来了看地址才决定放哪个货架；小文件问题=每个Reducer都在每个分区放一个小纸箱；文件合并=把小纸箱合成大箱子。

### 3.2 核心处理 process()（行885-970）

```java
public void process(Object row, int tag) throws HiveException {
    if (bDynParts) {
      // 1. 提取分区值
      ObjectInspectorUtils.partialCopyToStandardObject(dpWritables, row, dpStartCol, numDynParts, ...);
      for (Object o : dpWritables) {
        dpVals.add(o == null ? dpCtx.getDefaultPartitionName() : o.toString());
      }
      // 2. 获取/创建FSPaths
      fpaths = getDynOutPaths(dpVals, lbDirName);
      // 3. 只序列化数据列
      recordValue = serializer.serialize(row, subSetOI);
    }
    // 4. 写入
    rowOutWriters[0].write(recordValue);
}
```

### 3.3 小文件根因

```
N Tasks × M 分区 = N×M 个文件
100 Reducers × 200 分区 = 20000 个小文件！
```

### 3.4 合并机制

```
触发条件: avgSize < hive.merge.smallfiles.avgsize (16MB)
执行: MergeTask → CombineInput → MergeOperator
目标: hive.merge.size.per.task (256MB)
```

## 四、关键参数

| 参数名 | 默认值 | 说明 |

| `hive.exec.max.dynamic.partitions.pernode` | 100 | 单节点分区上限 |
| `hive.merge.mapredfiles` | false | MR后合并 |
| `hive.merge.sparkfiles` | false | Spark后合并 |

| `hive.merge.size.per.task` | 256MB | 目标文件大小 |

## 五、面试高频题

**Q1：为什么动态分区产生小文件？** A：N Tasks × M 分区 = N×M 个文件

**Q2：如何区分数据列和分区列？** A：dpStartCol 标记分界点，之前是数据列，之后是分区列

## 六、知识晶体

```
写入: row → 提取分区值 → getDynOutPaths → serialize(数据列) → write
小文件: N×M个 | 合并: avgSize<16MB触发MergeTask
关键: merge.sparkfiles=true + max.partitions.pernode
```
