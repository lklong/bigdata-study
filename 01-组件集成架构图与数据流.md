> 14 个组件之间的依赖关系、数据流向、集成方式

## 一、全局架构拓扑

```
                     用户 / 应用层 (Beeline / JDBC / REST)
                            │
               ┌────────────┴────────────┐
               ↓                         ↓
         Kyuubi (SQL网关)          Flink JM (流计算)
               │                         │
               ↓                         ↓
         Spark Engine              Flink TM
           │  │  │                   │  │
     ┌─────┘  │  └─────┐      ┌─────┘  └─────┐
     ↓        ↓        ↓      ↓              ↓
   HDFS    Hive HMS  Iceberg  Kafka         HBase
     │        │        │
     └────────┴────────┘
        底层存储 (HDFS / COS)

   资源调度: YARN (Container) 或 K8s (Pod)
   协调层:   ZK | Kerberos | Ranger | LDAP | Amoro
```

## 二、组件依赖矩阵

| 组件 | 依赖 | 如果依赖挂了 |
|------|------|-------------|
| Spark | YARN/K8s, HDFS, HMS, ZK(HA) | 无法提交/读写 |
| Hive HS2 | HMS, Tez/Spark, HDFS, ZK | 查询全失败 |
| HMS | MySQL/PG, HDFS | Spark/Flink 无法访问表 |
| Flink | YARN/K8s, HDFS(CP), Kafka, ZK(HA) | 作业失败 |
| Kyuubi | ZK, Spark/Flink Engine | 连接失败 |
| HDFS NN | ZK(HA), JN | HA failover 失败 |
| YARN RM | ZK(HA) | RM failover 失败 |
| ZK | **无外部依赖** | — |
| Kerberos | **无外部依赖** | 认证失败(已有Ticket暂不影响) |
| Ranger | MySQL, Solr/ES | 策略无法更新(缓存仍可鉴权) |

**关键: ZK 是最多组件依赖的底座，必须 5 节点 + 独立部署 + SSD**

## 三、数据流路径

### 批处理 ETL
```
COS/HDFS → Spark SQL (清洗) → Iceberg (存储) → Trino (查询)
认证: Kerberos → 授权: Ranger → 元数据: HMS
```

### 流处理
```
Kafka → Flink (处理) → Iceberg/HBase/Kafka
Checkpoint: TM → RocksDB → HDFS
```

### 交互查询
```
用户 → Kyuubi (ZK发现) → Spark Engine → HMS → HDFS/COS → 返回
```

## 五、集成配置速查

### Spark + Iceberg + HMS
```
spark.sql.catalog.spark_catalog=org.apache.iceberg.spark.SparkSessionCatalog
spark.sql.catalog.spark_catalog.type=hive
hive.metastore.uris=thrift://hms:9083
```

### Flink + Kafka + Iceberg
```sql
CREATE CATALOG ice WITH ('type'='iceberg','catalog-type'='hive','uri'='thrift://hms:9083');
CREATE TABLE ks (...) WITH ('connector'='kafka','topic'='t','properties.bootstrap.servers'='kafka:9092');
INSERT INTO ice.db.t SELECT * FROM ks;
```

### Kyuubi + Spark + Ranger
```
kyuubi.engine.type=SPARK_SQL
spark.sql.extensions=...RangerSparkExtension
spark.ranger.admin.url=http://ranger:6080
```
