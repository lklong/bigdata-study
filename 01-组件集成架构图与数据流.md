## 二、组件间依赖矩阵

| 组件 | 依赖 | 依赖方式 | 如果依赖挂了会怎样 |
|------|------|---------|-------------------|
| **Spark** | YARN/K8s, HDFS, HMS, ZK(HA) | 资源调度 + 数据读写 + 元数据 | 无法提交任务/无法读写数据 |
| **Hive HS2** | HMS, Tez/Spark, HDFS, ZK(发现) | 编译+执行+存储 | 查询全部失败 |
| **Hive HMS** | MySQL/PG, HDFS | 元数据存储 | Spark/Flink/Trino 全部无法访问表 |
| **Flink** | YARN/K8s, HDFS(CP), Kafka(Source), ZK(HA) | 资源+存储+消息+协调 | 作业失败/CP 失败 |
| **Kyuubi** | ZK(发现), Spark/Flink(引擎) | 服务发现+SQL执行 | 连接失败 |
| **HDFS NN** | ZK(HA), JN(EditLog) | 高可用 | HA failover 失败 |
| **HDFS DN** | NN(注册) | 块管理 | 数据不可读写 |
| **YARN RM** | ZK(HA) | 高可用 | RM failover 失败 |
| **YARN NM** | RM(心跳), HDFS(日志聚合) | 资源管理 | Container 无法启动 |
| **ZooKeeper** | **无外部依赖** | 自包含 | — |
| **Kerberos KDC** | **无外部依赖** | 自包含 | 所有认证失败（已有 Ticket 暂不受影响） |
| **Ranger Admin** | MySQL/PG, Solr/ES(审计) | 策略存储+审计 | 策略无法更新（Plugin 缓存仍可鉴权） |
| **Ranger Plugin** | Ranger Admin(策略拉取) | 定期拉策略 | 用本地缓存继续鉴权 |
| **LDAP** | **无外部依赖** | 自包含 | 新登录失败（已有会话不受影响） |
| **Iceberg** | HMS(Catalog), HDFS/COS(数据) | 元数据+存储 | 表不可访问 |
| **Amoro AMS** | ZK(可选), Iceberg/Hive 表 | 表管理 | 自优化停止（不影响读写） |

### 关键发现：**ZK 是最关键的单点依赖**

ZK 被以下组件依赖：HDFS HA, YARN HA, Kafka, HBase, Kyuubi, Flink HA, Amoro
→ **ZK 挂 = 大面积故障**
→ 必须 5 节点 + 独立部署 + SSD

## 三、数据流路径

### 3.1 批处理 ETL 链路

```
COS/HDFS (原始数据)
  → Spark SQL (清洗/转换)
    → Iceberg 表 (结果存储)
      → Trino/Spark (即席查询)

元数据链路:
  Hive Metastore ← Spark (注册表) ← Iceberg (管理元数据)
  
认证链路:
  Kerberos KDC → Spark (kinit) → HDFS (Service Ticket) → COS
  
授权链路:
  Ranger Admin → Spark Plugin (策略) → 检查用户是否可访问表/列
```

### 3.2 流处理链路

```
Kafka (实时数据)
  → Flink (流处理/窗口/Join)
    → Iceberg / HBase / Kafka (结果)

Checkpoint 链路:
  Flink TM → State (RocksDB) → Checkpoint (HDFS)
  
恢复链路:
  Flink JM → 读取 HDFS 上的 Checkpoint → 分发到 TM → 从断点继续
```

### 3.3 交互式查询链路

```
用户 (beeline/JDBC)
  → Kyuubi Server (认证 + 路由)
    → Spark Engine (SQL 执行)
      → HMS (获取表信息)
        → HDFS/COS (读取数据)
          → 返回结果
          
连接池化:
  Kyuubi Engine Pool (预热 N 个 Engine) → 用户秒级连接
```

### 3.4 数据湖治理链路

```
Amoro AMS (监控表状态)
  → 检测小文件/碎片分区
    → 调度 Optimizer (Spark/Flink)
      → 读取 Iceberg 表元数据
        → 合并小文件 → 写新数据文件
          → 更新 Iceberg Metadata
            → 过期旧快照 → 清理孤立文件
```

## 五、集成配置速查

### 5.1 Spark 读 Iceberg 表 (通过 HMS)

```properties
spark.sql.catalog.spark_catalog=org.apache.iceberg.spark.SparkSessionCatalog
spark.sql.catalog.spark_catalog.type=hive
spark.sql.extensions=org.apache.iceberg.spark.extensions.IcebergSparkSessionExtensions
hive.metastore.uris=thrift://hms:9083
```

### 5.2 Flink 读 Kafka 写 Iceberg

```sql
CREATE CATALOG iceberg_catalog WITH (
  'type'='iceberg', 'catalog-type'='hive', 'uri'='thrift://hms:9083'
);
CREATE TABLE kafka_source (...) WITH (
  'connector'='kafka', 'topic'='events', 'properties.bootstrap.servers'='kafka:9092'
);
INSERT INTO iceberg_catalog.db.events SELECT * FROM kafka_source;
```

### 5.3 Kyuubi + Spark + Ranger

```properties
# kyuubi-defaults.conf
kyuubi.engine.type=SPARK_SQL
spark.sql.extensions=org.apache.kyuubi.plugin.spark.authz.ranger.RangerSparkExtension
spark.ranger.admin.url=http://ranger:6080
spark.ranger.service.name=hive_prod
```

### 5.4 全链路 Kerberos

```properties
# core-site.xml (所有组件共用)
hadoop.security.authentication=kerberos
hadoop.security.authorization=true

# 各组件各自的 principal + keytab
# HDFS: hdfs/_HOST@REALM
# YARN: yarn/_HOST@REALM
# Hive: hive/_HOST@REALM
# Spark: 用户 principal (kinit 或 --principal)
```
