# 跨项目同构对比 — L4 源码深度分析

> **段位目标**：L4 创造者/科学家 — 类比迁移法
> **方法论**：源码精通九大法则之"类比迁移" — 跨项目找同构关系，一通百通
> **对比对象**：Kyuubi vs HiveServer2 vs Flink SQL Gateway vs Trino Gateway

---

## 白话版导读（费曼 L1）

> 这四个项目做的是**同一件事**：在计算引擎之上提供 SQL 网关服务。
>
> 类比：都是"翻译官"，把客户的 SQL 需求翻译给后端引擎。
> - **Kyuubi** = 全能翻译社（支持 Spark/Flink/Trino/Hive，多协议，多租户隔离）
> - **HiveServer2** = 老牌翻译（只支持 Hive/Spark/Tez，内嵌执行，单租户）
> - **Flink SQL Gateway** = 流处理专家（专注 Flink，会话集群，REST API）
> - **Trino Gateway** = 反向代理（查询级路由，无状态，负载均衡）

# 一、五维同构对比

## 1.1 架构模式

| 维度 | Kyuubi | HiveServer2 | Flink SQL Gateway | Trino Gateway |
|------|--------|-------------|-------------------|---------------|
| **架构类型** | 无状态网关 + 独立 Engine 进程 | 内嵌执行（Server=Engine） | 无状态网关 + Session 集群 | 反向代理 |
| **计算进程** | 独立 Spark/Flink/Trino 进程 | 内嵌 Tez/Spark SessionState | Flink Session/Per-Job 集群 | Trino Worker 集群 |
| **Server 是否参与计算** | 否（纯路由） | 是（内嵌执行引擎） | 否（纯路由） | 否（纯代理） |
| **Server 无状态** | 是（状态在 ZK + MetadataStore） | 否（Session 在内存） | 是（状态在 Flink 集群） | 是（无状态多副本） |
| **水平扩展** | 多 Server + ZK | 多 HS2（Session 不迁移） | 多 Gateway | 多 Gateway |

```
Kyuubi:
  Client → Server (无状态) → Engine (有状态，独立进程)
                                     ↓
                               Spark/Flink/Trino

HiveServer2:
  Client → HS2 (有状态，内嵌执行)
                    ↓
              Tez/Spark (内嵌)

Flink SQL Gateway:
  Client → Gateway (无状态) → Flink Session Cluster (有状态)

Trino Gateway:
  Client → Gateway (无状态，反向代理) → Trino Cluster
```

## 1.2 Session 管理

| 维度 | Kyuubi | HiveServer2 | Flink SQL Gateway | Trino Gateway |
|------|--------|-------------|-------------------|---------------|
| **Session 粒度** | 客户端连接级 | 客户端连接级 | 客户端连接级 | 查询级（无 Session） |
| **Session 存储** | ConcurrentHashMap | ConcurrentHashMap | 内存 + Flink State | 无 |
| **Session 隔离** | 多级（CONNECTION/USER/GROUP/SERVER） | 单级（每连接一个 SessionState） | 每连接一个 Flink Session | 无隔离（共享集群） |
| **空闲检测** | opHandleSet 引用计数 | lastAccessTime | Session TTL | N/A |
| **限流** | 三维度（user/ip/user+ip） | 无内置 | 无内置 | 基于路由规则 |
| **Session 迁移** | 不支持（绑定 Engine） | 不支持（内嵌状态） | 支持（Flink State 可恢复） | N/A |

**核心差异**：
- Kyuubi 的 Session 是**有状态绑定**（绑定到特定 Engine），一旦绑定不可迁移
- Flink SQL Gateway 的 Session 状态在 Flink 集群中，Gateway 本身无状态
- Trino Gateway 完全无 Session 概念，每个查询独立路由

## 1.3 引擎/集群管理

| 维度 | Kyuubi | HiveServer2 | Flink SQL Gateway | Trino Gateway |
|------|--------|-------------|-------------------|---------------|
| **引擎创建** | 按需 spark-submit | 内嵌（随 HS2 启动） | 预部署 Flink 集群 | 预部署 Trino 集群 |
| **引擎发现** | ZK 临时节点 | N/A（内嵌） | Flink REST API | HTTP 健康检查 |
| **引擎共享** | 多级 ShareLevel | 全部共享（单实例） | Session 集群共享 | 集群级共享 |
| **引擎池** | 支持（POLLING/RANDOM） | 不支持 | 支持（多 Session 集群） | 多集群路由 |
| **分布式锁** | ZK InterProcessMutex | 不需要 | 不需要 | 不需要 |
| **引擎生命周期** | 自动（idle/max timeout） | 随 HS2（手动管理） | 随 Flink 集群 | 随 Trino 集群 |
| **多引擎类型** | Spark/Flink/Trino/Hive/JDBC | Tez/Spark（内嵌） | 仅 Flink | 仅 Trino |

**Kyuubi 的独特优势**：
1. **按需创建引擎**：无需预部署，资源按需分配
2. **多级共享**：从完全隔离（CONNECTION）到完全共享（SERVER）
3. **多引擎类型**：同一网关接入 Spark + Flink + Trino

## 1.4 HA 高可用

| 维度 | Kyuubi | HiveServer2 | Flink SQL Gateway | Trino Gateway |
|------|--------|-------------|-------------------|---------------|
| **服务发现** | ZK/Etcd 临时节点 | ZK 持久+临时节点 | K8s Service | 无（配置文件/DNS） |
| **故障检测** | ZK session 超时 | ZK session 超时 | K8s 健康检查 | HTTP 健康检查 |
| **故障恢复** | Server 重启 + 批处理恢复 | Session 丢失 | Flink Checkpoint 恢复 | 查询重试 |
| **多实例负载均衡** | ZK 随机/轮询选择 | ZK 随机选择 | K8s LoadBalancer | 路由规则 |
| **脑裂处理** | ZK CP 模型 | ZK CP 模型 | K8s 自动处理 | 无状态无影响 |

## 1.5 认证授权

| 维度 | Kyuubi | HiveServer2 | Flink SQL Gateway | Trino Gateway |
|------|--------|-------------|-------------------|---------------|
| **认证** | 多认证链（Kerberos/LDAP/JDBC/Custom） | SASL（Kerberos/LDAP/Custom） | REST Token | OAuth2/Kerberos |
| **授权** | Ranger Spark Extension | Ranger Hive Plugin | SQL 权限 | Trino 内置 ACL |
| **代理用户** | spark-submit --proxy-user | doAs 机制 | 无 | Trino 用户映射 |
| **凭证管理** | CredentialManager 自动刷新 | DelegationToken Manager | 无 | 无 |

# 三、架构演化趋势

## 3.1 从内嵌到分离

```
2012: HS2 (内嵌执行) — Server = Engine
2019: Kyuubi 0.x (分离执行) — Server ≠ Engine
2023: Flink SQL Gateway (分离路由) — Gateway ≠ Cluster
趋势: 网关层越来越薄，计算层越来越独立
```

## 3.2 从有状态到无状态

```
HS2: Session 在 Server 内存 — 有状态，不可迁移
Kyuubi: Session 在 Server 内存，Engine 状态在 ZK — 半无状态
Flink Gateway: Session 状态在 Flink 集群 — 网关无状态
Trino Gateway: 无 Session — 完全无状态
趋势: 网关无状态化，状态外置到协调服务或计算集群
```

## 3.3 从单引擎到多引擎

```
HS2: 仅 Hive/Tez/Spark (内嵌)
Kyuubi: Spark/Flink/Trino/Hive/JDBC/Chat (6 种)
趋势: 统一 SQL 网关接入多种计算引擎，即 "Lakehouse Gateway"
```

# 四、选型建议

| 场景 | 推荐 | 理由 |
|------|------|------|
| Spark 为主的数据湖 | **Kyuubi** | 多级引擎共享 + 按需创建 + Ranger 授权 |
| 纯 Hive 存量系统 | HS2 | 简单，与 Hive 深度集成 |
| Flink 实时查询 | Flink SQL Gateway | 原生 Flink Session 支持 |
| Trino 联邦查询 | Trino Gateway | 无状态路由 + 多集群负载均衡 |
| 多引擎混合查询 | **Kyuubi** | 唯一支持 Spark+Flink+Trino 统一网关 |
