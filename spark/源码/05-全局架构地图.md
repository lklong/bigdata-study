## 四、SQL 执行流水线（QueryExecution）— 最核心的链路

```
SQL 文本 / DataFrame API
    ↓ (SparkSession.sql() / Dataset.xxx())
LogicalPlan (parsed)
    ↓ Analyzer.executeAndCheck()           ← 语义分析：解析表名、列名、类型推断
analyzed: LogicalPlan
    ↓ commandExecuted                      ← INSERT 等写入命令提前执行
    ↓ planNormalizationRules               ← 计划规范化（命中缓存）
normalized: LogicalPlan
    ↓ cacheManager.useCachedData()         ← 缓存替换
withCachedData: LogicalPlan
    ↓ Optimizer.executeAndTrack()          ← 逻辑优化：谓词下推、常量折叠、列裁剪
optimizedPlan: LogicalPlan
    ↓ SparkPlanner.plan()                  ← 物理计划生成
sparkPlan: SparkPlan
    ↓ preparations 规则链:
    │  ├── InsertAdaptiveSparkPlan (AQE)
    │  ├── CoalesceBucketsInJoin
    │  ├── PlanDynamicPruningFilters
    │  ├── PlanSubqueries
    │  ├── EnsureRequirements (Shuffle 插入)
    │  ├── ReplaceHashWithSortAgg
    │  ├── CollapseCodegenStages (全阶段代码生成)
    │  └── ReuseExchangeAndSubquery
executedPlan: SparkPlan
    ↓ executedPlan.execute()
toRdd: RDD[InternalRow]
```

所有计划节点继承自 **TreeNode**（`catalyst/.../trees/TreeNode.scala`），提供 transform/collect 等树操作。

## 五、Shuffle 系统架构

### 写入路径（Map 端）— SortShuffleManager 三路分发

| ShuffleHandle 类型 | 条件 | Writer | 特点 |
|---|---|---|---|
| `BypassMergeSortShuffleHandle` | 分区少 + 无 map-side combine | `BypassMergeSortShuffleWriter` | 每个分区一个文件，最后合并 |
| `SerializedShuffleHandle` | 无 combine + 序列化器支持重定位 + 分区 ≤ 16M | `UnsafeShuffleWriter` | 序列化排序，8字节指针，cache 友好 |
| `BaseShuffleHandle` | 兜底 | `SortShuffleWriter` → `ExternalSorter` | 反序列化排序，支持 combine |

### 读取路径（Reduce 端）

```
BlockStoreShuffleReader.read()
    ↓ ShuffleBlockFetcherIterator  ← 拉取远程/本地 Shuffle Block（流量控制: maxBytesInFlight/5）
    ↓ 反序列化为 K-V 迭代器
    ↓ Aggregator (如有)           ← combine 操作
    ↓ ExternalSorter (如有排序)   ← 排序 + Spill 到磁盘
    ↓ InterruptibleIterator        ← 支持任务取消
```

## 六、调度系统三层架构

```
RDD Action (count/collect/save)
    ↓
DAGScheduler                      ← Stage 级别
├── 切分 Stage (Shuffle 边界)
├── 提交 TaskSet
├── 处理 Shuffle 输出丢失
└── 重新提交失败 Stage
    ↓
TaskSchedulerImpl                 ← Task 级别
├── 延迟调度 (数据本地性)
├── 推测执行
├── Task 失败重试
└── 资源分配 (FIFO/Fair)
    ↓
SchedulerBackend                  ← 集群通信
├── StandaloneSchedulerBackend
├── YarnClientSchedulerBackend
└── YarnClusterSchedulerBackend
    ↓
Executor                          ← 任务执行
├── Task.run()
├── ShuffleWriter.write()
└── BlockManager (数据存取)
```
