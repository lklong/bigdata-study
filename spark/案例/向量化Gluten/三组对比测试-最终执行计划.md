## 🔧 测试关键参数（每组对齐）

### 资源（按集群最大资源拉满）

```
集群: 3节点 × 32core/115GB = 96core/345GB
单 executor: 5 cores × 9g heap × 2g overhead = 11GB
Driver: 2 cores × 2g
maxExecutors: 9（共 45 cores + 99GB heap，留余量给 driver/AM）
```

### Native vs Gluten 对齐（保证公平）

```
Native:
  --num-executors 9 --executor-cores 5 --executor-memory 9g
  --conf spark.executor.memoryOverhead=2g
  --conf spark.plugins=                                             # 关 Gluten
  --conf spark.shuffle.manager=...SortShuffleManager
  --conf spark.memory.offHeap.enabled=false

Gluten:
  --num-executors 9 --executor-cores 5 --executor-memory 5g          # heap 减小
  --conf spark.executor.memoryOverhead=1g
  --conf spark.memory.offHeap.enabled=true
  --conf spark.memory.offHeap.size=8g                                # offHeap 给 Velox
  # 总内存: 5g+1g+8g = 14g/exec ≈ Native 11g+少量 GC overhead，等量对比
  --conf spark.plugins=org.apache.gluten.GlutenPlugin
  --conf spark.shuffle.manager=...ColumnarShuffleManager
  --conf spark.gluten.sql.columnar.backend.lib=velox
  --conf spark.gluten.memory.allocator=jemalloc
  --conf spark.gluten.sql.columnar.backend.velox.spillEnabled=false
```

### 通用必加（Hive 2.3.7 / 3.1.3 兼容）

```
--conf spark.sql.adaptive.enabled=true
--conf spark.sql.hive.convertMetastoreParquet=true
--conf spark.sql.parquet.enableVectorizedReader=true
--conf spark.sql.shuffle.partitions=128
--conf spark.dynamicAllocation.enabled=false                         # 关动态分配
--conf spark.shuffle.service.enabled=false
--conf spark.executorEnv.JAVA_HOME=$KONA_JDK                         # 透传 Kona JDK
--conf spark.yarn.appMasterEnv.JAVA_HOME=$KONA_JDK
--conf spark.driver.extraJavaOptions=$ADD_OPENS                      # JDK11+ 必备
--conf spark.executor.extraJavaOptions=$ADD_OPENS
```
