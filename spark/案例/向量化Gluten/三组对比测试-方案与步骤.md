# 三组对比测试方案 — Spark + Gluten 跨集群跨版本

> **测试时间**：2026-05-24
> **数据**：cosn://lkl-bj-update-1308597516/meson/data/bench/ （5表 / 29亿行 / 91.7GB）
> **测试规模**：每组 7 个 Q × 2 遍取稳定值
> **测试原则**：纯净测试（不改集群 JDK / so 全带 / Kona JDK 统一）

## 二、测试原则（大哥铁律）

1. ✅ **纯净测试**：不改集群 `JAVA_HOME` 全局环境，不改 `/etc/profile`，不动系统默认 java 路径
2. ✅ **so 文件全带上**：Gluten/Velox 依赖的 native lib 通过 jar 内嵌 + `--jars` 提交
3. ✅ **JDK 统一用 Kona JDK**：临时 `JAVA_HOME=` 仅作用于当前 spark-submit 命令
4. ✅ **资源对齐**：Native vs Gluten 使用相同 `--num-executors / --executor-cores`（Gluten heap 减小、offHeap 加大，总内存等量）
5. ✅ **关键参数对齐**：`spark.sql.adaptive.enabled=true` / `spark.sql.shuffle.partitions=128` / `spark.dynamicAllocation.enabled=false`
6. ✅ **结果稳定性**：每个 Q 跑 2 遍取最小值（去掉冷启动影响）

## 五、关键 Spark 参数（每组对齐）

### Native（关 Gluten）
```
--num-executors 9 --executor-cores 5 --executor-memory 9g
--conf spark.executor.memoryOverhead=2g
--conf spark.plugins=                                              # 显式关 Gluten
--conf spark.shuffle.manager=org.apache.spark.shuffle.sort.SortShuffleManager
--conf spark.memory.offHeap.enabled=false
```

### Gluten（开 Velox）
```
--num-executors 9 --executor-cores 5 --executor-memory 5g
--conf spark.executor.memoryOverhead=1g
--conf spark.plugins=org.apache.gluten.GlutenPlugin
--conf spark.gluten.sql.columnar.backend.lib=velox
--conf spark.shuffle.manager=org.apache.spark.shuffle.sort.ColumnarShuffleManager
--conf spark.memory.offHeap.enabled=true
--conf spark.memory.offHeap.size=8g                                # heap 5g+offHeap 8g+overhead 1g = 14g
--conf spark.gluten.memory.allocator=jemalloc
--conf spark.gluten.sql.columnar.backend.velox.spillEnabled=false  # 避开 1.5 sumVector bug
--conf spark.gluten.sql.columnar.backend.velox.aggregationSpillEnabled=false
--conf spark.gluten.sql.columnar.backend.velox.joinSpillEnabled=false
```

### 通用（不能少）
```
--conf spark.sql.adaptive.enabled=true
--conf spark.sql.hive.convertMetastoreParquet=true                 # Hive 兼容必加
--conf spark.sql.parquet.enableVectorizedReader=true
--conf spark.sql.shuffle.partitions=128
--conf spark.shuffle.service.enabled=false
--conf spark.dynamicAllocation.enabled=false                       # 关动态分配，资源固定
--conf spark.executorEnv.JAVA_HOME=$KONA_JDK                       # ⭐ 透传 Kona JDK
--conf spark.yarn.appMasterEnv.JAVA_HOME=$KONA_JDK
--conf spark.driver.extraJavaOptions=$ADD_OPENS                    # ⭐ JDK11+ 必备
--conf spark.executor.extraJavaOptions=$ADD_OPENS
```

## 八、踩坑预案（来自历史经验）

| 坑 | 现象 | 解法 | 已在脚本里处理 |
|---|---|---|---|
| 1 | `UnsupportedOperationException: ETypeConverter` | `spark.sql.hive.convertMetastoreParquet=true` | ✅ |
| 2 | `VeloxUserError: sumVector != nullptr` | 关 Velox spill | ✅ |
| 3 | `NoSuchMethodError: ByteBuffer.flip()` | Kona JDK11+ + `add-opens` | ✅ |
| 4 | `Permission denied: user=root, /user` | 用 hadoop 用户跑 | ✅ |
| 5 | EMR Spark 与 Gluten 不兼容（HashAggregateExec） | 用社区版 Spark / 用配套 Gluten | ✅ |
| 6 | Worker 没 JDK | `spark.executorEnv.JAVA_HOME` 透传 + worker 本地装 Kona JDK | ⚠️ 需先确认 worker 有 |
| 7 | so 文件 not found | Gluten bundle jar 内嵌所有 .so，`--jars` 带上即可 | ✅ |
| 8 | cosn 鉴权失败 | core-site.xml 配 cosn AK/SK | ⚠️ 集群已配 |
