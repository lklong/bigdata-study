1. Gluten 1.6.0 预编译包从 `downloads.apache.org/gluten/1.6.0/apache-gluten-1.6.0-bin-spark-3.5.tar.gz` 获取

## 四、踩坑记录

| # | 问题 | 根因 | 解决 |
|---|---|---|---|
| 1 | Gluten 1.6.0 + JDK 8 启动报 `UnsupportedClassVersionError (61.0)` | 预编译 jar 用 JDK 17 编译 | 安装 Kona JDK 17 |
| 2 | 执行时 SIGSEGV crash `ConnectLibHdfs failed` | Velox 找不到 libhdfs.so | 设置 `LD_LIBRARY_PATH=$HADOOP_HOME/lib/native` |
| 3 | AQE 开启时 EXPLAIN 看不到 Columnar 节点 | AQE 延迟物化物理计划 | `spark.sql.adaptive.enabled=false` |
| 4 | Spark 3.5.3 + Gluten 1.6.0 复杂 SQL 报 `NoSuchMethodError: HashAggregateExec.copy$default$9()` | Gluten 1.6.0 编译目标是 Spark 3.5.5+，3.5.3 API 不兼容 | 升级到 Spark 3.5.8 |
| 5 | YARN executor exit code 50 | Worker 节点没有 JDK 17 | 所有节点 `yum install java-17-konajdk` |
| 6 | YARN 报 `Required memory above max threshold (7373MB)` | executor 4g + offHeap 4g + overhead > 7.3G | 降低为 executor 2g + offHeap 3g |
| 7 | 远端无法访问 GitHub | 网络隔离 | 从 `downloads.apache.org` 下载或本地迅雷下载后 SFTP 上传 |

### YARN 模式阻塞问题（待解决）

Gluten 1.6.0 on YARN Executor 报错：
```
sun.misc.Unsafe or java.nio.DirectByteBuffer.<init>(long, int) not available
```

**根因分析**：
- Gluten 1.6.0 的 native code 回调 Java 时需要通过 `sun.misc.Unsafe` 创建 DirectByteBuffer
- JDK 17 模块系统默认不开放 `java.base/sun.misc` 给 unnamed module
- 需要 `--add-opens=java.base/sun.misc=ALL-UNNAMED` 参数
- **已确认**：通过 `spark.executor.extraJavaOptions` 传递此参数（无论命令行还是 defaults），Executor 仍报此错误
- **推测**：EMR YARN 的 container-executor 或 NM 配置中可能有参数覆盖/过滤机制

**可能的解决路径**：
1. 修改所有 NM 节点的 `yarn-site.xml` 添加 `yarn.nodemanager.admin-env`
2. 检查 EMR 是否有 `container-executor.cfg` 限制了 JVM 参数传递
3. 回退使用 Gluten 1.2.0（支持 JDK 8 的旧版本）
4. 从源码用 JDK 8 编译 Gluten（需 64GB 内存机器）

### 待完成（下次继续）

## 六、关键配置速查

```bash
# 环境变量
export SPARK_HOME=/opt/spark-poc
export JAVA_HOME=/usr/lib/jvm/TencentKona-17.0.18.b1
export HADOOP_HOME=/usr/local/service/hadoop
export LD_LIBRARY_PATH=$HADOOP_HOME/lib/native:$JAVA_HOME/lib/server:$JAVA_HOME/lib

# Gluten on YARN 命令模板
$SPARK_HOME/bin/spark-sql --master yarn \
  --deploy-mode client \
  --num-executors 6 \
  --executor-cores 2 \
  --executor-memory 2g \
  --driver-memory 2g \
  --conf "spark.plugins=org.apache.gluten.GlutenPlugin" \
  --conf "spark.memory.offHeap.enabled=true" \
  --conf "spark.memory.offHeap.size=3g" \
  --conf "spark.shuffle.manager=org.apache.spark.shuffle.sort.ColumnarShuffleManager" \
  --conf "spark.driver.extraJavaOptions=--add-opens=java.base/java.nio=ALL-UNNAMED --add-opens=java.base/sun.nio.ch=ALL-UNNAMED --add-opens=java.base/java.lang=ALL-UNNAMED --add-opens=java.base/java.util=ALL-UNNAMED --add-opens=java.base/java.lang.invoke=ALL-UNNAMED" \
  --conf "spark.executor.extraJavaOptions=--add-opens=java.base/java.nio=ALL-UNNAMED --add-opens=java.base/sun.nio.ch=ALL-UNNAMED --add-opens=java.base/java.lang=ALL-UNNAMED --add-opens=java.base/java.util=ALL-UNNAMED --add-opens=java.base/java.lang.invoke=ALL-UNNAMED" \
  --conf "spark.executor.extraLibraryPath=/usr/local/service/hadoop/lib/native" \
  --conf "spark.executorEnv.HADOOP_HOME=/usr/local/service/hadoop" \
  --conf "spark.executorEnv.JAVA_HOME=/usr/lib/jvm/TencentKona-17.0.18.b1" \
  --conf "spark.sql.adaptive.enabled=false" \
  --conf "spark.sql.shuffle.partitions=50" \
  -e "YOUR_SQL"
```
